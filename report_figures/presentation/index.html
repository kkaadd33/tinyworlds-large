Tiny World Model on Zelda

Fixing action-model collapse · adding controllability · sharper images
— at zero added parameters

Khaled Almansoori · MBZUAI · TinyWorlds (Genie reproduction)

What I changed — in plain terms

1
The action model was collapsing

The part that learns the controls died during training (it kept shrinking to nothing).

Fix: much lower learning rate + a "variance band" that keeps it alive + an entropy loss that forces it to use all its action codes. → trains stably, uses all 4 actions.

2
Predictions ignored the action

The future frames barely responded to the control input — not really playable.

Fix: train sometimes WITHOUT the action (action-dropout) + classifier-free guidance at inference + stronger action injection (FiLM). → the action now steers the world.

3
Images were blurry

Plain pixel loss averages detail away → mushy frames.

Fix: a perceptual loss (frozen VGG) during training only. → sharper textures, 0 extra parameters at inference.
